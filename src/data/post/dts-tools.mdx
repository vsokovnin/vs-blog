**Инструменты стратегического планирования** — это набор практических методов для каждого этапа разработки стратегии: от исследования клиентов до валидации бизнес-модели. По данным [IDEO](https://designthinking.ideo.com/), компании, системно использующие инструменты дизайн-мышления, в 2 раза быстрее выводят продукты на рынок и на 50% чаще достигают product-market fit.

- 12 ключевых инструментов DTS, сгруппированных по этапам процесса

## Карта инструментов по процессам DTS

Инструменты **дизайн-мышления для стратегии (Design Thinking for Strategy, DTS)** распределены по четырём ключевым процессам методологии.

```
ПРОЦЕСС Н (Наблюдение) — дивергентный, прошлое
├── Персона
├── Customer Decision Journey
├── Journey Map
├── Мышление вслух
└── Пять почему

ПРОЦЕСС И (Изучение) — конвергентный, прошлое
├── Карта эмпатии
└── Jobs-to-be-Done Canvas

ПРОЦЕСС П (Проектирование) — дивергентный, будущее
├── Брейнсторминг
├── Антиконвенциональное мышление
├── Business Model Canvas
└── Value Proposition Canvas

ПРОЦЕСС В (Валидация) — конвергентный, будущее
├── Карта допущений
├── Дерево DuPont
└── Эксперименты
```

Согласно статье «[Концепция и инструменты фасилитации](https://cyberleninka.ru/article/n/kontseptsiya-i-instrumenty-fasilitatsii-dlya-upravleniya-konfliktnymi-protivorechiyami-zainteresovannyh-storon-pri-transformatsii)» (Митенков, 2024), правильный выбор инструментов фасилитации повышает эффективность стратегических сессий при трансформации.

## Инструменты процесса Н: Наблюдение

Процесс наблюдения направлен на сбор инсайтов о клиентах дивергентным способом — без фильтрации и оценки.

### Персона (Persona)

**Цель:** Создать человечное описание целевого клиента для принятия решений.

**Когда использовать:** В начале процесса Н, после определения стратегического фокуса на клиентах.

**Компоненты персоны:**

| Блок | Содержание | Источник данных |
|------|------------|-----------------|
| **Демография** | Имя, возраст, профессия, доход | Аналитика, CRM |
| **Jobs-to-be-Done** | Функциональные, эмоциональные, социальные задачи | Интервью |
| **Боли (Pains)** | Что мешает достичь цели | Интервью, поддержка |
| **Выгоды (Gains)** | Что хочет получить | Интервью |
| **Цитата** | Типичное высказывание | Вербатим из интервью |
| **Поведение** | Как принимает решения | Наблюдение |

**Шаблон персоны:**

```
┌─────────────────────────────────────────────────────────┐
│  ПЕРСОНА: [Имя]                              [Фото]    │
├─────────────────────────────────────────────────────────┤
│  Возраст: ___    Профессия: ___    Доход: ___          │
├─────────────────────────────────────────────────────────┤
│  JOBS-TO-BE-DONE                                        │
│  • Функциональные: ________________________________     │
│  • Эмоциональные: _________________________________     │
│  • Социальные: ____________________________________     │
├─────────────────────────────────────────────────────────┤
│  БОЛИ                        │  ВЫГОДЫ                  │
│  1. ___________________      │  1. ___________________  │
│  2. ___________________      │  2. ___________________  │
│  3. ___________________      │  3. ___________________  │
├─────────────────────────────────────────────────────────┤
│  ЦИТАТА: «_________________________________________»    │
├─────────────────────────────────────────────────────────┤
│  ПОВЕДЕНИЕ: ________________________________________    │
└─────────────────────────────────────────────────────────┘
```

**Типичная ошибка:** Создавать персону на основе предположений, а не данных. Персона без интервью — это фантазия, а не инструмент.

### Customer Decision Journey

**Цель:** Отобразить путь клиента от осознания потребности до лояльности.

**Когда использовать:** После создания персоны, для понимания точек контакта.

**Этапы Customer Decision Journey:**

```
ТРИГГЕР → ОСОЗНАНИЕ → ОЦЕНКА → ПОКУПКА → ОПЫТ → ЛОЯЛЬНОСТЬ
    │          │          │         │        │         │
    ▼          ▼          ▼         ▼        ▼         ▼
 Что          Как       Какие    Что      Что      Что
 запускает   узнаёт    критерии влияет   формирует возвращает
 поиск?      о нас?    выбора?  на       опыт?     клиента?
                                решение?
```

**Для каждого этапа определить:**
- Действия клиента
- Мысли и эмоции
- Точки контакта (touchpoints)
- Барьеры и драйверы
- Наши возможности

### Journey Map

**Цель:** Детально отобразить опыт клиента с эмоциональной кривой.

**Когда использовать:** Для глубокого анализа конкретного сценария использования.

**Структура Journey Map:**

| Этап | Действия | Мысли | Эмоции | Touchpoints | Возможности |
|------|----------|-------|--------|-------------|-------------|
| 1 | Что делает | Что думает | 😀😐😟 | Где контакт | Как улучшить |
| 2 | ... | ... | ... | ... | ... |

**Эмоциональная кривая:** Визуализируйте изменение эмоций клиента на протяжении пути. Точки с негативными эмоциями — приоритеты для улучшения.

### Мышление вслух (Think Aloud)

**Цель:** Пассивное наблюдение за клиентом с вербализацией его мыслей.

**Когда использовать:** Для понимания реального поведения, а не декларируемого.

**Протокол проведения:**

1. **Подготовка.** Определите задачу для клиента (например, «забронируйте услугу»).
2. **Инструктаж.** Попросите клиента проговаривать все мысли вслух.
3. **Наблюдение.** Не вмешивайтесь, только фиксируйте.
4. **Фиксация.** Записывайте действия, высказывания, паузы, ошибки.

**Что фиксировать:**

| Время | Действие | Высказывание | Эмоция | Примечание |
|-------|----------|--------------|--------|------------|
| 0:30 | Открыл меню | «Так, где тут услуги...» | Замешательство | Навигация неочевидна |

### Пять почему (5 Whys)

**Цель:** Определить корневую причину проблемы через последовательные «почему».

**Когда использовать:** Когда нужно понять истинную мотивацию или причину поведения.

**Пример применения:**

```
Проблема: Клиенты не возвращаются после первого визита.

Почему 1: Потому что забывают о нас.
Почему 2: Потому что мы не напоминаем.
Почему 3: Потому что нет системы напоминаний.
Почему 4: Потому что CRM слишком сложная.
Почему 5: Потому что CRM создана для корпораций, не для нас.

Корневая причина: Отсутствие подходящего инструмента для микробизнеса.
```

**Правило:** Не останавливайтесь на симптоме. Копайте до корневой причины.

## Инструменты процесса И: Изучение

Процесс изучения трансформирует инсайты из наблюдения в структурированные знания.

### Карта эмпатии (Empathy Map)

**Цель:** Структурировать понимание клиента по четырём квадрантам.

**Когда использовать:** После наблюдений, для синтеза инсайтов в команде.

**Шаблон карты эмпатии:**

```
┌─────────────────────────────────────────────────────────┐
│                      ДУМАЕТ И ЧУВСТВУЕТ                 │
│         Что действительно важно? Главные заботы?        │
│         ____________________________________________    │
├────────────────────────┬────────────────────────────────┤
│       СЛЫШИТ           │           ВИДИТ                │
│  Что говорят друзья,   │  Что видит в окружении?        │
│  коллеги, рынок?       │  Что предлагают конкуренты?    │
│  ____________________  │  __________________________    │
├────────────────────────┴────────────────────────────────┤
│                      ГОВОРИТ И ДЕЛАЕТ                   │
│         Поведение на публике. Внешний вид.              │
│         ____________________________________________    │
├────────────────────────┬────────────────────────────────┤
│        БОЛИ            │          ВЫГОДЫ                │
│  Страхи, разочарования │  Желания, критерии успеха      │
│  ____________________  │  __________________________    │
└────────────────────────┴────────────────────────────────┘
```

### Jobs-to-be-Done Canvas

**Цель:** Структурировать понимание задач, для выполнения которых клиент «нанимает» продукт.

**Когда использовать:** Для глубокого понимания мотивации покупки.

По данным [Strategyn](https://strategyn.com/jobs-to-be-done/), компании, использующие JTBD-подход, достигают 86% успешности в инновациях против 17% в среднем по рынку.

**Структура JTBD:**

| Компонент | Вопрос | Пример (CRM для салонов) |
|-----------|--------|--------------------------|
| **Job** | Какую работу выполняет? | Не терять клиентов |
| **Контекст** | В какой ситуации? | После первого визита |
| **Мотивация** | Зачем это важно? | Повторные визиты = 70% выручки |
| **Ожидаемый результат** | Как выглядит успех? | Клиент возвращается через месяц |
| **Текущее решение** | Как решает сейчас? | Excel, блокнот, память |
| **Барьеры текущего решения** | Почему не работает? | Забывает, нет напоминаний |

## Инструменты процесса П: Проектирование

Процесс проектирования направлен на генерацию и оформление идей бизнес-модели.

### Брейнсторминг

**Цель:** Генерировать большое количество идей без оценки.

**Когда использовать:** В начале процесса П, для расширения пространства решений.

**Правила брейнсторминга (по IDEO):**

1. **Отложить оценку.** Никакой критики во время генерации.
2. **Стремиться к количеству.** 100 идей лучше 10 «хороших».
3. **Развивать чужие идеи.** «Да, и...» вместо «Да, но...».
4. **Приветствовать безумные идеи.** Они часто содержат зерно инновации.
5. **Визуализировать.** Рисуйте, используйте стикеры.
6. **Один разговор.** Не перебивать, не делиться на группы.
7. **Фокус на теме.** Возвращаться к вопросу.

**Формат проведения:**

| Этап | Длительность | Действие |
|------|--------------|----------|
| Разминка | 5 мин | Простое творческое упражнение |
| Генерация | 15-20 мин | Записать максимум идей на стикеры |
| Кластеризация | 10 мин | Сгруппировать похожие идеи |
| Голосование | 5 мин | Каждый ставит 3 точки на лучшие идеи |

### Антиконвенциональное мышление

**Цель:** Генерировать разрушительные (disruptive) идеи через переворачивание допущений.

**Когда использовать:** Когда брейнсторминг даёт только инкрементальные идеи.

**Алгоритм антиконвенционального мышления:**

1. **Выписать допущения отрасли.** Что все считают истиной?
   - Пример: «CRM должна иметь веб-интерфейс»

2. **Перевернуть каждое допущение.**
   - Переворот: «А если CRM будет только в мессенджере?»

3. **Разработать идею на основе переворота.**
   - Идея: WhatsApp-native CRM для микробизнеса

**Шаблон антиконвенционального мышления:**

| Допущение отрасли | Переворот | Новая идея |
|-------------------|-----------|------------|
| «Клиенты хотят много функций» | «Клиенты хотят минимум функций» | Одна функция, но идеально |
| «Нужен отдел продаж» | «Продукт продаёт себя сам» | Product-Led Growth |

### Business Model Canvas

**Цель:** Визуализировать и проектировать бизнес-модель на одной странице.

**Когда использовать:** Для оформления идеи в структурированную бизнес-модель.

По данным [Strategyzer](https://www.strategyzer.com/library/the-business-model-canvas), Business Model Canvas используется более 5 миллионов раз и является стандартом для стартапов и корпоративных инноваций.

**9 блоков Business Model Canvas:**

```
┌───────────────┬───────────────┬───────────────┬───────────────┬───────────────┐
│   КЛЮЧЕВЫЕ    │   КЛЮЧЕВЫЕ    │   ЦЕННОСТНОЕ  │  ОТНОШЕНИЯ С  │   СЕГМЕНТЫ    │
│   ПАРТНЁРЫ    │  АКТИВНОСТИ   │  ПРЕДЛОЖЕНИЕ  │   КЛИЕНТАМИ   │   КЛИЕНТОВ    │
│               │               │               │               │               │
│  Кто помогает │  Что делаем   │  Какую        │  Как строим   │  Для кого     │
│  создавать    │  для создания │  ценность     │  отношения    │  создаём      │
│  ценность?    │  ценности?    │  создаём?     │               │  ценность?    │
│               ├───────────────┤               ├───────────────┤               │
│               │   КЛЮЧЕВЫЕ    │               │    КАНАЛЫ     │               │
│               │   РЕСУРСЫ     │               │               │               │
│               │               │               │  Как доставля-│               │
│               │  Что нужно    │               │  ем ценность? │               │
│               │  для работы?  │               │               │               │
├───────────────┴───────────────┼───────────────┴───────────────┴───────────────┤
│        СТРУКТУРА ЗАТРАТ       │              ПОТОКИ ДОХОДОВ                    │
│                               │                                               │
│  Какие главные затраты?       │  За что платят клиенты?                       │
└───────────────────────────────┴───────────────────────────────────────────────┘
```

**Порядок заполнения:**

1. Начните с **Сегментов клиентов** — для кого создаёте ценность
2. Затем **Ценностное предложение** — какую проблему решаете
3. Далее **Каналы** и **Отношения** — как доставляете ценность
4. Потом **Потоки доходов** — как зарабатываете
5. Затем **Ключевые ресурсы**, **Активности**, **Партнёры** — что нужно для работы
6. В конце **Структура затрат** — сколько это стоит

### Value Proposition Canvas

**Цель:** Детализировать связь между ценностным предложением и потребностями клиента.

**Когда использовать:** Для глубокой проработки fit между продуктом и клиентом.

**Структура Value Proposition Canvas:**

```
┌─────────────────────────────────┐    ┌─────────────────────────────────┐
│      ЦЕННОСТНОЕ ПРЕДЛОЖЕНИЕ     │    │        ПРОФИЛЬ КЛИЕНТА          │
│                                 │    │                                 │
│  ┌─────────────────────────┐    │    │    ┌─────────────────────────┐  │
│  │     СОЗДАТЕЛИ ВЫГОД     │    │    │    │        ВЫГОДЫ           │  │
│  │                         │◄───┼────┼───►│                         │  │
│  │  Как создаём выгоды?    │    │    │    │  Чего хочет клиент?     │  │
│  └─────────────────────────┘    │    │    └─────────────────────────┘  │
│                                 │    │                                 │
│  ┌─────────────────────────┐    │    │    ┌─────────────────────────┐  │
│  │   ПРОДУКТЫ И УСЛУГИ     │    │    │    │    ЗАДАЧИ КЛИЕНТА       │  │
│  │                         │◄───┼────┼───►│                         │  │
│  │  Что предлагаем?        │    │    │    │  Что делает клиент?     │  │
│  └─────────────────────────┘    │    │    └─────────────────────────┘  │
│                                 │    │                                 │
│  ┌─────────────────────────┐    │    │    ┌─────────────────────────┐  │
│  │    ОБЕЗБОЛИВАТЕЛИ       │    │    │    │         БОЛИ            │  │
│  │                         │◄───┼────┼───►│                         │  │
│  │  Как снимаем боли?      │    │    │    │  Что мешает клиенту?    │  │
│  └─────────────────────────┘    │    │    └─────────────────────────┘  │
└─────────────────────────────────┘    └─────────────────────────────────┘
```

**Критерий качества:** Каждый элемент слева должен адресовать конкретный элемент справа. Если продукт не решает боль и не создаёт выгоду — это feature creep.

## Инструменты процесса В: Валидация

Процесс валидации тестирует гипотезы бизнес-модели в реальных условиях.

### Карта допущений (Assumption Map)

**Цель:** Идентифицировать и приоритизировать допущения для тестирования.

**Когда использовать:** Перед проектированием экспериментов.

**Матрица приоритизации допущений:**

```
                    ВАЖНОСТЬ ДЛЯ УСПЕХА
                    Низкая          Высокая
               ┌────────────────────────────────┐
      Высокая  │   Тестировать    │  ТЕСТИРОВАТЬ │
               │   при наличии    │   В ПЕРВУЮ   │
НЕОПРЕДЕЛЁН-   │   ресурсов       │   ОЧЕРЕДЬ    │
НОСТЬ          ├──────────────────┼──────────────┤
               │   Отложить       │  Тестировать │
      Низкая   │                  │   быстро     │
               └────────────────────────────────┘
```

**Правило:** Начинайте с допущений в правом верхнем квадранте — они могут убить всю стратегию.

### Дерево DuPont

**Цель:** Проанализировать финансовую жизнеспособность бизнес-модели.

**Когда использовать:** Для валидации финансового компонента стратегии.

**Структура дерева DuPont:**

```
                         ROE
                          │
          ┌───────────────┼───────────────┐
          │               │               │
    Рентабельность   Оборачиваемость   Финансовый
      продаж          активов          рычаг
          │               │               │
    Чистая прибыль   Выручка        Активы
    ─────────────    ─────────      ─────────
      Выручка        Активы        Капитал
```

**Применение в DTS:**

Для каждого элемента бизнес-модели определите влияние на дерево DuPont:
- **Ценностное предложение** → Цена → Выручка
- **Структура затрат** → Себестоимость → Рентабельность
- **Каналы** → Затраты на привлечение → Маржа
- **Сегменты** → Объём рынка → Потенциал выручки

### Эксперименты

**Цель:** Протестировать допущения в реальных условиях с минимальными затратами.

**Когда использовать:** После приоритизации допущений.

**Типы экспериментов по уровню достоверности:**

| Тип | Достоверность | Затраты | Пример |
|-----|---------------|---------|--------|
| **Интервью** | Низкая | Низкие | «Вы бы использовали это?» |
| **Лендинг** | Средняя | Низкие | Измерить % конверсии в заявку |
| **Консьерж** | Высокая | Средние | Выполнить услугу вручную |
| **MVP** | Высокая | Высокие | Минимальный работающий продукт |
| **Пилот** | Очень высокая | Высокие | Реальные продажи реальным клиентам |

**Шаблон эксперимента:**

```
ЭКСПЕРИМЕНТ #___

ДОПУЩЕНИЕ: ________________________________________________

ГИПОТЕЗА: Мы верим, что [целевая аудитория] будет [действие],
          потому что [причина].

МЕТОД: [ ] Интервью  [ ] Лендинг  [ ] Консьерж  [ ] MVP  [ ] Пилот

КРИТЕРИЙ УСПЕХА: Гипотеза подтверждена, если _______________
                 Гипотеза опровергнута, если _______________

РАЗМЕР ВЫБОРКИ: ________    СРОК: ________

РЕЗУЛЬТАТ:  [ ] Подтверждено  [ ] Опровергнуто  [ ] Неопределённо

СЛЕДУЮЩИЙ ШАГ: ____________________________________________
```

## Практический пример: инструменты для B2B SaaS

Продолжим пример CRM для салонов красоты из всей серии.

**Процесс Н — Наблюдение:**

1. **Персона:** Анна, 35 лет, владелица салона на 3 мастера, доход 150К руб./мес.
2. **Journey Map:** Путь от записи клиента до повторного визита.
3. **5 Whys:** Почему клиенты не возвращаются → нет напоминаний → CRM сложны.

**Процесс И — Изучение:**

1. **Карта эмпатии:** Анна думает о выручке, видит конкурентов с CRM, боится потерять клиентов.
2. **JTBD:** «Не терять клиентов» — функциональная задача. «Чувствовать контроль» — эмоциональная.

**Процесс П — Проектирование:**

1. **Антиконвенциональное мышление:** Допущение «CRM = веб-интерфейс» → Переворот «CRM в WhatsApp».
2. **BMC:** Сегмент = микросалоны, ПЦП = «База клиентов в мессенджере», Доход = 790 руб./мес.
3. **VPC:** Боль «забываю позвонить» → Обезболиватель «автонапоминание».

**Процесс В — Валидация:**

1. **Карта допущений:** Критическое допущение — «Владельцы готовы платить за простую CRM».
2. **Эксперимент:** Лендинг + реклама → 5% конверсия в заявку.
3. **DuPont:** CAC = 2000 руб., LTV = 9500 руб., LTV/CAC = 4,75 — модель работает.

## Типичные ошибки при работе с инструментами

### Ошибка 1: Инструмент без цели

**Проблема:** Команда заполняет Business Model Canvas, потому что «так надо».

**Последствия:** Формальное упражнение без инсайтов. Canvas заполнен, но ничего не изменилось.

**Решение:** Перед использованием инструмента сформулируйте вопрос, на который ищете ответ.

### Ошибка 2: Заполнение из головы

**Проблема:** Персону, Journey Map, карту эмпатии заполняют на основе предположений.

**Последствия:** Фантазия вместо данных. Решения на основе ложных предпосылок.

**Решение:** Каждый инсайт должен быть подкреплён цитатой из интервью или данными наблюдения.

### Ошибка 3: Один раз и забыть

**Проблема:** BMC заполнили на воркшопе и больше не обновляли.

**Последствия:** Артефакт устаревает. Реальность ушла вперёд, а команда работает по старой модели.

**Решение:** Пересматривайте ключевые инструменты после каждого цикла валидации.

### Ошибка 4: Слишком много инструментов

**Проблема:** Команда использует все 12 инструментов для каждого проекта.

**Последствия:** Паралич анализа. Время уходит на артефакты, а не на решения.

**Решение:** Выбирайте 3-5 инструментов, релевантных текущему этапу и вопросу.

## Чеклист выбора инструментов

**Для этапа Наблюдение (Н):**
- [ ] Персона — если фокус на клиентах
- [ ] Journey Map — если важен опыт использования
- [ ] 5 Whys — если нужно понять корневую причину

**Для этапа Изучение (И):**
- [ ] Карта эмпатии — для синтеза инсайтов в команде
- [ ] JTBD Canvas — для понимания мотивации покупки

**Для этапа Проектирование (П):**
- [ ] Брейнсторминг — для расширения пространства решений
- [ ] Антиконвенциональное мышление — если нужны disruptive идеи
- [ ] BMC — для структурирования бизнес-модели
- [ ] VPC — для детализации ценностного предложения

**Для этапа Валидация (В):**
- [ ] Карта допущений — для приоритизации тестов
- [ ] Эксперименты — для проверки гипотез
- [ ] Дерево DuPont — для финансовой валидации

## Часто задаваемые вопросы

### Какие инструменты использовать в первую очередь?

Начните с Персоны и Business Model Canvas — они дают базовое понимание клиента и бизнес-модели. Остальные инструменты добавляйте по мере возникновения конкретных вопросов. Не используйте все 12 инструментов сразу — это ведёт к параличу анализа.

### Можно ли использовать инструменты без фасилитатора?

Да, но эффективность снижается. Фасилитатор помогает команде не уходить в споры, соблюдать тайминг и фиксировать результаты. Если фасилитатора нет — назначьте одного участника ответственным за процесс, а не за контент.

### Как часто обновлять артефакты?

Персону и BMC — после каждого значимого инсайта из интервью или эксперимента. Journey Map — при изменении продукта или процесса. Минимум: пересмотр всех артефактов раз в квартал.

### Какой инструмент для B2B, какой для B2C?

Все инструменты применимы и в B2B, и в B2C. Различается глубина: в B2B персона включает роль в закупке (инициатор, ЛПР, пользователь), в B2C — акцент на эмоциональных jobs-to-be-done.

## Выводы

- **Инструменты — это средство, а не цель.** Перед использованием сформулируйте вопрос, на который ищете ответ. Canvas ради canvas — пустая трата времени.

- **Данные важнее предположений.** Каждый инсайт в персоне, Journey Map, карте эмпатии должен быть подкреплён реальными данными — цитатой из интервью или результатом наблюдения.

- **Выбирайте инструменты под этап.** Для наблюдения — персона и Journey Map. Для проектирования — BMC и VPC. Для валидации — эксперименты. Не используйте всё сразу.

- **Артефакты живут и обновляются.** Бизнес-модель меняется после каждого эксперимента. Пересматривайте ключевые артефакты минимум раз в квартал.
