**Процесс разработки стратегии по методологии DTS** — это структурированная последовательность из десяти этапов, объединённых в три уровня, с контрольными точками для принятия решений. В отличие от линейного стратегического планирования, DTS использует итеративный подход: команда проходит циклы наблюдения, изучения, проектирования и валидации, возвращаясь к предыдущим этапам при необходимости.

По данным [McKinsey](https://www.mckinsey.com/capabilities/strategy-and-corporate-finance/our-insights/have-you-tested-your-strategy-lately), только 35% стратегий проходят четыре или более стратегических теста — и этот показатель снизился до 21% в 2024-2025 годах. Структурированный процесс с чёткими критериями на каждом этапе повышает вероятность создания работающей стратегии.

## Карта процесса DTS

Процесс дизайн-мышления для стратегии состоит из десяти этапов, сгруппированных в три уровня. Каждый уровень отвечает на конкретный стратегический вопрос и завершается контрольной точкой, где лица, принимающие решения, утверждают или отклоняют результаты.

```
СТРАТЕГИЧЕСКИЙ БРИФ
        │
        ▼
┌───────────────────────────────────────────────────┐
│           ФУНДАМЕНТАЛЬНЫЙ УРОВЕНЬ                 │
│                                                   │
│   Б:Бриф → С:Среда → Т:Тренды → Ф:Фокус          │
│                                                   │
│   ══► КОНТРОЛЬНАЯ ТОЧКА 1                        │
└───────────────────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────────────────┐
│           УРОВЕНЬ БИЗНЕС-МОДЕЛИ                   │
│                                                   │
│        ┌──────────────────────────┐               │
│        │         ┌────┐           │               │
│        ▼         │    ▼           │               │
│   Н:Наблюд. → И:Изуч. → П:Проект. → В:Валид.     │
│        ▲                          │               │
│        └──────────────────────────┘               │
│                                                   │
│   ══► КОНТРОЛЬНАЯ ТОЧКА 2                        │
└───────────────────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────────────────┐
│           УРОВЕНЬ КОНКУРЕНЦИИ                     │
│                                                   │
│   К:Конкуренция → КС:Коммуникация                │
│                                                   │
│   ══► КОНТРОЛЬНАЯ ТОЧКА 3                        │
└───────────────────────────────────────────────────┘
        │
        ▼
   РЕАЛИЗАЦИЯ СТРАТЕГИИ
```

**Десять этапов процесса DTS:**
1. **Б — Стратегический бриф:** определение границ и целей проекта
2. **С — Анализ среды:** исследование текущей конкурентной среды
3. **Т — Анализ трендов:** идентификация факторов изменений
4. **Ф — Выбор фокуса:** определение измерения для конкурентного преимущества
5. **Н — Наблюдение:** сбор инсайтов о клиентах
6. **И — Изучение:** синтез инсайтов в знания
7. **П — Проектирование:** создание прототипов бизнес-модели
8. **В — Валидация:** тестирование ключевых допущений
9. **К — Конкурентное позиционирование:** анализ реакций конкурентов
10. **КС — Коммуникация стратегии:** формулирование и донесение стратегии

Согласно исследованию [КиберЛенинки о стратегировании в условиях неопределённости](https://cyberleninka.ru/article/n/strategirovanie-v-usloviyah-neopredelennosti-tseli-i-instrumenty), итеративный подход к стратегическому планированию позволяет компаниям быстрее адаптироваться к изменениям среды. В отличие от линейного процесса «анализ → планирование → реализация», DTS предполагает многократные циклы на уровне бизнес-модели.

## Три уровня процесса

### Фундаментальный уровень: где конкурировать?

Первый уровень отвечает на вопрос «где мы будем конкурировать?» и включает четыре этапа:

| Этап | Вопрос | Результат |
|------|--------|-----------|
| **Бриф** | Каковы границы и цели проекта? | Стратегический бриф с руководящими принципами |
| **Среда** | В какой среде мы сейчас работаем? | Карта клиентских сегментов, конкурентов, отраслевых сил |
| **Тренды** | Какие факторы изменят отрасль? | Анализ клиентских, структурных, инновационных и внешних трендов |
| **Фокус** | На чём сосредоточить усилия? | Выбор одного из четырёх измерений: клиенты, предложения, компетенции, финансы |

По данным [McKinsey](https://www.mckinsey.com/capabilities/strategy-and-corporate-finance/our-insights/the-strategic-yardstick-you-cant-afford-to-ignore), компании, использующие стратегическое планирование с предиктивной аналитикой, реагируют на изменения рынка на 40% быстрее и демонстрируют на 25% более высокую прибыльность.

На фундаментальном уровне команда работает с **упрощённой бизнес-моделью** из четырёх компонентов: клиенты, предложения, компетенции, финансы. Задача — выбрать одно измерение для создания конкурентного преимущества.

### Уровень бизнес-модели: как создавать ценность?

Второй уровень отвечает на вопрос «как мы будем создавать ценность?» и представляет собой ядро дизайн-мышления в DTS:

| Этап | Тип мышления | Фокус во времени | Результат |
|------|--------------|------------------|-----------|
| **Наблюдение** | Дивергентное | Прошлое | Инсайты о клиентах и их jobs-to-be-done |
| **Изучение** | Конвергентное | Прошлое | Структурированные знания, паттерны, противоречия |
| **Проектирование** | Дивергентное | Будущее | Прототипы бизнес-модели (15 элементов) |
| **Валидация** | Конвергентное | Будущее | Проверенные допущения, готовая бизнес-модель |

| Традиционный подход | Дизайн-мышление для стратегии |
|---------------------|-------------------------------|
| Анализ данных → решение | Наблюдение → гипотеза → эксперимент → решение |
| Один цикл планирования | Многократные итерации Н→И→П→В |
| Допущения принимаются на веру | Допущения валидируются экспериментами |
| Фокус на прошлых данных | Баланс прошлого (Н, И) и будущего (П, В) |

Этот уровень итеративен: после валидации команда может вернуться к наблюдению для сбора дополнительных данных или к проектированию для корректировки бизнес-модели. Стрелки на диаграмме показывают возможные возвраты.

### Уровень конкуренции: как победить?

Третий уровень отвечает на вопрос «как мы победим?» и включает два этапа:

| Этап | Вопрос | Инструменты |
|------|--------|-------------|
| **Конкурентное позиционирование** | Как мы выиграем у конкурентов? | Пять вопросов Портера, теория игр, равновесие Нэша |
| **Коммуникация стратегии** | Как донести стратегию до стейкхолдеров? | Стратегическая история, KPI, план коммуникации |

На этом уровне применяется теория игр для анализа возможных реакций конкурентов. Команда определяет устойчивое конкурентное преимущество — такое, которое конкуренты не смогут легко скопировать или нейтрализовать.

## Контрольные точки

Контрольные точки — это формальные моменты принятия решений между уровнями процесса. На каждой контрольной точке лица, принимающие решения (Decision Takers), утверждают результаты этапа или отправляют команду на доработку. Это обеспечивает управляемость процесса и снижает риски масштабных ошибок.

### Контрольная точка 1: выбор стратегического фокуса

| Аспект | Описание |
|--------|----------|
| **Расположение** | После этапа Ф (Выбор фокуса) |
| **Что утверждается** | Стратегический фокус: клиенты, предложения, компетенции или финансы |
| **Кто решает** | Лица, принимающие решения (Decision Takers) |
| **Критерии прохождения** | Фокус обоснован анализом среды и трендов, соответствует руководящим принципам из брифа |
| **Если не пройден** | Возврат к этапу Ф для пересмотра или к С/Т для дополнительного анализа |

### Контрольная точка 2: утверждение бизнес-модели

| Аспект | Описание |
|--------|----------|
| **Расположение** | После этапа В (Валидация) |
| **Что утверждается** | Детализированная бизнес-модель (15 элементов), результаты валидации |
| **Кто решает** | Лица, принимающие решения (Decision Takers) |
| **Критерии прохождения** | Ключевые допущения валидированы; желательность, выполнимость, жизнеспособность подтверждены |
| **Если не пройден** | Возврат к циклу Н→И→П→В для итерации |

### Контрольная точка 3: финальное утверждение стратегии

| Аспект | Описание |
|--------|----------|
| **Расположение** | После этапа КС (Коммуникация стратегии) |
| **Что утверждается** | Полная стратегия: фокус + бизнес-модель + конкурентное позиционирование + KPI |
| **Кто решает** | Лица, принимающие решения (Decision Takers) |
| **Критерии прохождения** | Пять вопросов Портера отвечены, конкурентное преимущество устойчиво, стратегическое сообщение валидировано |
| **Если не пройден** | Возврат к этапу К или даже к уровню бизнес-модели |

**Ошибка:** Разные люди принимают решения на разных контрольных точках.

**Решение:** Зафиксируйте состав Decision Takers в стратегическом брифе. Одни и те же люди должны участвовать во всех трёх контрольных точках — это обеспечивает преемственность решений и ответственность за конечный результат.

## Роли в процессе DTS

Дизайн-мышление для стратегии определяет четыре категории участников стратегического проекта. Каждая роль имеет чёткую ответственность и степень вовлечённости на разных этапах.

### Лица, принимающие решения (Decision Takers)

**Кто:** Члены совета директоров, CEO, топ-менеджмент.

**Ответственность:** Утверждение результатов на контрольных точках, определение руководящих принципов стратегии.

**Критическое требование:** Решения никогда не делегируются на более низкий уровень. Голосование — анти-паттерн, ведущий к посредственным решениям.

### Разработчики стратегии (Strategy Designers)

**Кто:** Команда из 5-10 старших стратегов, включая представителей лиц, принимающих решения.

**Ответственность:** Создание и валидация стратегии на всех этапах процесса.

**Лучшая практика:** Не делегировать ключевую работу внешним консультантам — команда должна владеть результатом.

### Эксперты (Experts)

**Кто:** Люди с опытом из разных отраслей, дающие свежий взгляд.

**Ответственность:** Помощь на этапах наблюдения, изучения и проектирования.

**Критическое требование:** Выбирать экспертов ДО начала процесса, чтобы избежать смещения выборки.

### Поддержка процесса (Process Supporters)

**Кто:** Фасилитаторы, коучи, менеджеры проекта.

**Ответственность:** Управление процессом, документирование результатов, координация коммуникации.

**Рекомендация:** Привлекать 1-2 независимых внешних стратегических коуча для объективности.

### Матрица вовлечённости ролей

| Роль | Б | С | Т | Ф | Н | И | П | В | К | КС | КТ |
|------|---|---|---|---|---|---|---|---|---|----|----|
| **Decision Takers** | ◐ | ○ | ○ | ● | ○ | ○ | ○ | ● | ○ | ● | ● |
| **Strategy Designers** | ● | ● | ● | ● | ● | ● | ● | ● | ● | ● | ● |
| **Experts** | ○ | ◐ | ◐ | ◐ | ● | ● | ● | ◐ | ◐ | ○ | ○ |
| **Process Supporters** | ● | ● | ● | ● | ● | ● | ● | ● | ● | ● | ● |

● — активное участие, ◐ — частичное участие, ○ — информирование, КТ — контрольные точки

## Десять типичных рисков

На основе практического опыта применения методологии выделяется десять наиболее распространённых рисков при разработке стратегии. Для каждого риска определена митигация — действия, которые снижают вероятность или последствия проблемы.

| # | Риск | Этап | Митигация |
|---|------|------|-----------|
| 1 | **Решения на контрольных точках пересматриваются** | КТ 1-3 | Фиксировать решения письменно, обеспечивать присутствие одних и тех же Decision Takers |
| 2 | **Лица, принимающие решения, меняются во время процесса** | Б → КС | Определить и зафиксировать Decision Takers в стратегическом брифе |
| 3 | **Ключевые члены команды уходят без передачи знаний** | Весь процесс | Документировать инсайты, проводить регулярные сессии обмена знаниями |
| 4 | **Не достигнуто согласие по стратегическому фокусу** | Ф | Использовать структурированный процесс выбора, избегать голосования |
| 5 | **Слишком много времени на наблюдение, слишком мало на проектирование** | Н, П | Устанавливать временные рамки, применять правило 80/20 |
| 6 | **Допущения не валидируются («мы знаем лучше»)** | В | Культура экспериментирования, независимые коучи |
| 7 | **Неправильная приоритизация допущений для валидации** | В | Начинать с допущений, которые могут опровергнуть всю стратегию |
| 8 | **Валидации слишком долгие и дорогие** | В | Быстрые, дешёвые эксперименты; не искать чрезмерную точность |
| 9 | **Допущения продвигаются, а не тестируются** | В | Искать опровержения, не подтверждения; независимые тестировщики |
| 10 | **Конкуренты и их реакции игнорируются** | К | Обязательный анализ теории игр на уровне конкуренции |

Согласно [Harvard Business Review](https://hbr.org/2017/11/many-strategies-fail-because-theyre-not-actually-strategies), многие документы, которые компании называют «стратегиями», на самом деле представляют собой списки целей или финансовые планы. Настоящая стратегия должна отвечать на вопрос «как мы победим?», а не просто «чего мы хотим достичь?».

**Ошибка:** Команда проводит много интервью и наблюдений (этап Н), но не остаётся времени на проектирование альтернативных бизнес-моделей (этап П).

**Решение:** Установите временные рамки для каждого этапа заранее. Применяйте правило 80/20: 80% ценности приходит от 20% наблюдений. Лучше начать проектирование раньше и вернуться к наблюдению при необходимости.

## Шесть принципов процесса DTS

Помимо структуры этапов и ролей, процесс дизайн-мышления для стратегии опирается на шесть ключевых принципов:

1. **Клиентоцентричность** — потребности клиентов в центре всех решений
2. **Итеративность** — улучшение через многократные циклы Н→И→П→В
3. **Прототипирование** — быстрое создание и тестирование идей до масштабных инвестиций
4. **Валидация** — тестирование гипотез в реальных условиях, а не принятие на веру
5. **Just-in-time** — анализ только того, что нужно для следующего решения
6. **Fail fast** — быстрые неудачи ведут к быстрому успеху

Принцип **just-in-time** особенно важен для управления бюджетом. Вместо детального планирования всего процесса заранее, команда фокусируется на следующей контрольной точке и уточняет план после каждого этапа.

## Часто задаваемые вопросы

### Сколько времени занимает полный процесс DTS?

Типичный проект дизайн-мышления для стратегии занимает 3-6 месяцев. Фундаментальный уровень (бриф, среда, тренды, фокус) — 4-6 недель. Уровень бизнес-модели (наблюдение, изучение, проектирование, валидация) — 6-12 недель с несколькими итерациями. Уровень конкуренции — 2-4 недели.

### Можно ли пропустить некоторые этапы процесса?

Пропускать этапы не рекомендуется — каждый строится на результатах предыдущего. Однако глубина проработки может варьироваться. Для небольших компаний или локальных стратегий отдельные этапы могут быть сокращены. Критично сохранить все три контрольные точки для обеспечения качества решений.

### Чем контрольные точки DTS отличаются от stage-gate в традиционном планировании?

Контрольные точки DTS предполагают возможность итераций — команда может вернуться к предыдущим этапам для доработки. В традиционном stage-gate процесс линеен: после прохождения «ворот» возврат не предусмотрен. DTS также требует, чтобы одни и те же люди принимали решения на всех контрольных точках.

### Как распределить роли, если команда маленькая?

В маленьких компаниях один человек может совмещать несколько ролей. CEO часто выступает и как Decision Taker, и как Strategy Designer. Важно сохранить разделение на контрольных точках: тот, кто разрабатывал решение, не должен сам его утверждать без внешней проверки.

## Выводы

- **Десять этапов, три уровня, три контрольные точки.** Структурированный процесс DTS обеспечивает управляемость и снижает риски провала стратегии.
- **Итеративность на уровне бизнес-модели.** Циклы наблюдение→изучение→проектирование→валидация позволяют тестировать гипотезы до масштабных инвестиций.
- **Фиксируйте Decision Takers в брифе.** Одни и те же люди должны принимать решения на всех контрольных точках — это критично для преемственности и ответственности.
