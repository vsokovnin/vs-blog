**Design Thinking for Strategy (DTS)**, или дизайн-мышление для стратегии, — это методология разработки корпоративной стратегии, объединяющая человекоцентричный подход дизайн-мышления, фреймворк бизнес-модели и теорию игр для конкурентного анализа. В отличие от традиционного стратегического планирования, DTS начинает с глубокого понимания клиентов и итеративно тестирует гипотезы вместо того, чтобы полагаться на предположения.

По данным [McKinsey Design Index](https://www.mckinsey.com/capabilities/mckinsey-design/our-insights/the-business-value-of-design), компании-лидеры по дизайну демонстрируют на 32% более высокий рост выручки и на 56% более высокую доходность для акционеров по сравнению с конкурентами. При этом [McKinsey](https://www.mckinsey.com/capabilities/strategy-and-corporate-finance/our-insights/why-do-most-transformations-fail-a-conversation-with-harry-robinson) отмечает, что 70% стратегических трансформаций терпят неудачу из-за отсутствия валидации ключевых допущений — именно эту проблему решает DTS.

## Когда традиционного планирования недостаточно

Классические методы стратегического планирования — SWOT-анализ, пять сил Портера, матрица BCG — остаются полезными инструментами. Они хорошо работают в стабильных отраслях с предсказуемой динамикой: данные из прошлого надёжно экстраполируются, конкуренты известны, потребности клиентов меняются медленно.

Однако в условиях высокой неопределённости — технологических разрывов, появления новых бизнес-моделей, быстро меняющихся потребительских предпочтений — аналитические методы сталкиваются с ограничениями:

**Четыре ситуации, где DTS дополняет традиционный подход:**

1. **Когда нужны новые идеи, а не только анализ.** Классические фреймворки помогают структурировать известное, но не генерируют принципиально новых решений. DTS добавляет этап синтеза: наблюдение за клиентами → генерация гипотез → прототипирование.

2. **Когда допущения критичны для успеха.** По данным [CB Insights](https://www.cbinsights.com/research/report/startup-failure-reasons-top/), 35% стартапов терпят неудачу из-за отсутствия рыночной потребности. DTS встраивает валидацию допущений в процесс до масштабных инвестиций.

3. **Когда клиенты меняются быстрее, чем данные о них.** Опросы и исследования рынка отражают прошлое. Этнографические методы DTS позволяют увидеть формирующиеся потребности, которых ещё нет в статистике.

4. **Когда стратегия должна вовлекать команду.** Традиционное планирование часто происходит «наверху» и спускается вниз. DTS предполагает участие кросс-функциональной команды, что повышает качество решений и упрощает внедрение.

DTS не заменяет аналитические инструменты, а дополняет их итеративным процессом: наблюдение → гипотезы → прототипирование → валидация → корректировка. Такой подход особенно ценен, когда цена ошибки высока, а будущее сложно предсказать.

## Три уровня методологии DTS

Методология DTS организована в три последовательных уровня, каждый из которых отвечает на конкретный стратегический вопрос:

| Уровень | Вопрос | Этапы | Результат |
|---------|--------|-------|-----------|
| **Фундаментальный** | Где мы будем конкурировать? | Бриф → Среда → Тренды → Фокус | Стратегический фокус |
| **Бизнес-модель** | Как мы будем создавать ценность? | Наблюдение → Изучение → Проектирование → Валидация | Валидированная бизнес-модель |
| **Конкуренция** | Как мы победим? | Позиционирование → Коммуникация | Устойчивое конкурентное преимущество |

Между уровнями расположены контрольные точки — формальные моменты, когда руководство принимает решение о продолжении, корректировке или остановке проекта. Это обеспечивает управляемость процесса и снижает риски масштабных ошибок.

## Ключевые концепции

| Термин | Определение |
|--------|-------------|
| Дизайн-мышление для стратегии (DTS) | Методология разработки стратегии, объединяющая дизайн-мышление, фреймворк бизнес-модели и теорию игр для создания валидированной стратегии с устойчивым конкурентным преимуществом |
| Стратегический фокус | Одно из четырёх измерений (клиенты, предложения, компетенции, финансы), по которому компания стремится превосходить конкурентов и создавать уникальную ценность |
| Контрольная точка (Checkpoint) | Формальный момент принятия решения между уровнями DTS, где лица, принимающие решения, утверждают или отклоняют результаты этапа |
| Jobs-to-be-done | Функциональные, эмоциональные и социальные задачи, которые клиент «нанимает» продукт или услугу выполнить |
| Упрощённая бизнес-модель | Модель из четырёх компонентов (клиенты, предложения, компетенции, финансы) для определения стратегического фокуса |
| Детализированная бизнес-модель | Расширенная модель из 15 элементов для полного описания механизма создания ценности |
| Валидация | Процесс тестирования ключевых допущений стратегии через быстрые и дешёвые эксперименты до масштабных инвестиций |

## Выводы

- **Стратегия = Фокус + Бизнес-модель + Позиционирование.** DTS даёт структуру для последовательной проработки всех трёх компонентов с валидацией на каждом этапе.
- **Начинайте с клиентов, а не с аналитики.** Дизайн-мышление переворачивает традиционный подход: сначала понимание jobs-to-be-done, потом стратегические решения.
- **Тестируйте допущения до масштабных инвестиций.** Быстрые эксперименты снижают риск провала стратегии и дают уверенность в решениях.
